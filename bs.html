Bubble Sort

Definition

Bubble Sort is a basic comparison-based sorting algorithm. It works by repeatedly stepping through the list, comparing adjacent elements, and swapping them if they are in the wrong order. This process is repeated until no swaps are needed, indicating that the list is sorted.

  • Bubble Sort is a simple sorting algorithm.
  • It works by repeatedly swapping adjacent elements if they are in the wrong order.
  • Smaller elements gradually "bubble" to the top of the list with each pass.
  • It is not suitable for large datasets due to high time complexity.
  • It's easy to understand and implement.
  • It is a stable sort — maintains the relative order of equal elements.
  • Works well for small or nearly sorted datasets.

Use Cases

  • Good for teaching and educational purposes.
  • Useful when the array is nearly sorted.
  • Helps demonstrate basic sorting logic.

Example

Consider the array [5, 3, 8, 4]:

  • Step 1: Compare 5 and 3 → Swap → [3, 5, 8, 4]
  • Step 2: Compare 5 and 8 → No Swap → [3, 5, 8, 4]
  • Step 3: Compare 8 and 4 → Swap → [3, 5, 4, 8]
  • Repeat the process until the array is fully sorted.

void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
  
class BubbleSort {
    public static void bubbleSort(int[] arr) {
        for (int i = 0; i < arr.length - 1; i++) {
            for (int j = 0; j < arr.length - i - 1; j++) {
                if (arr[j] > arr[j + 1]) {
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                }
            }
        }
    }
}
  
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]

Complexities

Type Time Meaning
Best Case O(n) Array is already sorted.
Average Case O(n2) Random order of elements.
Worst Case O(n2) Array is sorted in reverse order.
Space Complexity O(1) Sorting is done in-place.

Advantages

  • Very simple and easy to implement.
  • No additional memory required.
  • Stable sort (preserves order of equal elements).

Disadvantages

  • Not efficient for large datasets.
  • Time complexity is high in average and worst case.
  • Not used in practical applications for performance-critical tasks.
